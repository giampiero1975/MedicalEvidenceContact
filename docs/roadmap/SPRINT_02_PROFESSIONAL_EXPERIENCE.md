# Sprint 02 — Professional Experience

Status: **In progress**

## Obiettivo

Realizzare l'esperienza completa dell'utente Professional partendo dai mockup approvati, senza perdere le funzionalità già coperte dai test.

## Regola di lavoro

La UI può cambiare, il comportamento applicativo no. Ogni estrazione dalla dashboard deve mantenere compatibilità con i flussi esistenti finché la nuova pagina non è verificata.

## Schermate previste

1. Dashboard Professional
2. Esperienze e percorsi di studio
3. Documenti professionali
4. Moodle e formazione
5. Candidature
6. Colloqui
7. Profilo e sicurezza

## Sequenza di implementazione

- [x] Shell applicativa Professional
- [x] Dashboard Professional v1
- [ ] Pagina Esperienze e percorsi di studio
- [ ] Pagina Documenti professionali
- [ ] Pagina Candidature
- [ ] Allineamento Moodle e formazione
- [ ] Allineamento Colloqui
- [ ] Rifinitura Profilo e sicurezza
- [ ] Responsive finale
- [ ] Test Feature verdi

## Criteri di accettazione

- Navigazione laterale coerente con i mockup.
- Ogni sezione ha una responsabilità chiara.
- Nessuna funzionalità storica rimossa.
- Professional non può accedere alle aree Business o Admin.
- Tutti i test Feature passano con SQLite in-memory.
- Le pagine sono utilizzabili su desktop e mobile.

## Stato corrente

Il primo blocco operativo dello sprint è l'estrazione di **Esperienze e percorsi di studio** in una pagina dedicata, mantenendo temporaneamente la compatibilità della dashboard con i test preesistenti.
